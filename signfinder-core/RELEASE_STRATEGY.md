## ROADMAP — ПОРЯДОК РАБОТ

### Веха 1 — v1.20 Pre-flight Check (СЛЕДУЮЩАЯ)

Договорное ревью через LLM. Превращает утилиту в договорного агента.
Полная концепция — PREFLIGHT_CONCEPT.md.

Ключевое:
- LLM-first, без детерминированной базы (работает для всех языков)
- Промпт под язык/юрисдикцию (динамический)
- Информационный режим (не блокирует подпись)
- Два независимых светофора (подпись + ревью)
- Точки вывода: Streamlit → письмо агента → sandbox

После v1.20 — MVP функционально завершён.

### Веха 2 — Облако Azure + Landing + Sandbox

Платформа Azure (сознательный выбор — закрыть нулевую зону по Azure для
hands-on опыта: деплой + observability. Docker портируется провайдер-агностично).

Архитектура:
```
DOCKER (разработка локально):
  signfinder-cloud/          ← новый репо, снапшот core+api
  ├── signfinder-core/
  ├── signfinder-api/        (+ сессии, потом tenant+idempotency)
  ├── landing/               ← НОВЫЙ внешний интерфейс (не Streamlit)
  │   ├── hero video (loop, музыка+надписи, ru/en)
  │   └── sandbox-widget (Вариант 3: онбординг)
  └── docker-compose.yml

  Streamlit → остаётся как ПРОДУКТОВЫЙ sandbox для платящих (после v1.20)

AZURE (портирование после Docker):
  ├── API + core (Container Apps / ACI)
  ├── Landing (статика на Blob + CDN)
  ├── Storage (Blob, сессии сейчас / tenant потом)
  ├── Secrets (Key Vault)
  └── Analytics (PostHog)
```

Sandbox онбординг (Вариант 3):
1. Загрузи PDF → 2. Покажи найденные места (без подписи) →
3. Введи компанию+имя → 4. Нарисуй/загрузи подпись →
5. Покажи подписанный + замечания pre-flight

Сессии сейчас (sessionID, удаление через час), тенанты — отдельной волной
перед монетизацией.

### Веха 3 — Tenant + Idempotency (перед монетизацией)

- X-Tenant-ID, изоляция storage per-tenant
- Idempotency-Key на /analyze, /sign
- Key Vault для секретов (TD-06 блокер закрыт)
- Rate limiting per tenant
- Audit log

### Веха 4 — Монетизация

- Stripe (сначала Invoice links вручную, потом Checkout)

### Веха 5 — Платный трафик (по метрикам)

- LinkedIn Ads + Show HN / Product Hunt
- Только когда unit economics из Вехи 4 сходятся

## АНАЛИТИКА (PostHog, free tier)

Воронка onboarding (9 событий):
1. landing_view → 2. hero_video_played → 3. sandbox_started →
4. pdf_uploaded → 5. analysis_shown → 6. profile_filled →
7. signature_added → 8. result_shown → 9. contact_left

Технические метрики:
- analysis_latency_ms (p50/p95)
- step_duration (время на шаге онбординга)
- geo (страна по IP, без хранения сырого IP — GDPR)
- language (ru/en версия)
- referrer (Reddit/HN/LinkedIn/direct)
- device (desktop/mobile)

GDPR: сырой IP не хранить, документы не логировать, подписи не дольше сессии,
cookie consent (decline non-essential по умолчанию).

Ключевые отчёты:
- Воронка 9 шагов → точка отвала
- landing_view → contact_left (главная метрика спроса)
- referrer → качество трафика по площадкам
- ru vs en → конверсия по языку
