# SignFinder — Руководство по установке и использованию

SignFinder — система автоматического поиска мест подписи в договорах и проставления электронной подписи. Поддерживает PDF и DOCX, работает на вашем компьютере без передачи документов в облако (кроме запросов к языковой модели).

---

## Часть 1. Установка

### Шаг 1 — Установить Git

Git нужен для скачивания программы с GitHub.

1. Перейти на https://git-scm.com/download/win
2. Скачать установщик (64-bit) и запустить
3. Во всех шагах установщика нажимать **Next** — настройки по умолчанию подходят
4. Проверить установку: открыть **Командную строку** (Win+R → `cmd`) и ввести:
   ```
   git --version
   ```
   Должно появиться что-то вроде `git version 2.xx.x`

### Шаг 2 — Установить Docker Desktop

Docker — это среда, в которой запускается SignFinder. Вам не нужно ничего устанавливать отдельно — всё необходимое запускается внутри Docker.

1. Перейти на https://www.docker.com/products/docker-desktop/
2. Нажать **Download for Windows**
3. Запустить установщик, нажимать **Next**
4. После установки **перезагрузить компьютер**
5. Запустить Docker Desktop (значок кита в трее должен стать зелёным)
6. Проверить: открыть Командную строку и ввести:
   ```
   docker --version
   ```
   Должно появиться: `Docker version 2x.x.x`

> **Важно:** Docker Desktop должен быть запущен (значок в трее) каждый раз когда вы работаете с SignFinder.

### Шаг 3 — Скачать SignFinder

1. Создайте папку для программы, например `C:\SignFinder`
2. Откройте Командную строку и перейдите в созданную папку:
   ```
   cd C:\SignFinder
   ```
3. Скачайте SignFinder:
   ```
   git clone https://github.com/alexgeorg2507-creator/SignPDFMVPLocal.git .
   ```
   Точка в конце обязательна — означает «скачать в текущую папку».

### Шаг 4 — Создать файл настроек

В папке `C:\SignFinder` есть файл `.env.example` — это шаблон с настройками.

1. Скопируйте его:
   ```
   copy .env.example .env
   ```
2. Откройте файл `.env` любым текстовым редактором (например Блокнот или Notepad++)
3. Заполните **три обязательных поля**:

```
API_KEY=придумайте_длинный_пароль_минимум_20_символов
ACCESS_CODE=1234
```

**API_KEY** — внутренний пароль системы. Придумайте любую строку из букв и цифр длиной не менее 20 символов. Пример: `mySuperLocalKey_20260601_abc`. Этот пароль вы вводить не будете — он нужен для внутренней защиты.

**ACCESS_CODE** — код для входа в интерфейс SignFinder. Можете оставить `1234` или поменять на любой другой.

> LLM-ключ (для работы искусственного интеллекта) вводится через интерфейс программы после запуска — смотрите Шаг 6.

### Шаг 5 — Собрать и запустить

В Командной строке, в папке `C:\SignFinder`:

```
docker compose build
```

Первый запуск занимает **15–25 минут** — Docker скачивает все необходимые компоненты. Последующие запуски занимают секунды.

После успешной сборки запустите:

```
docker compose up -d
```

Флаг `-d` означает «запустить в фоне». Дождитесь пока появится сообщение что все сервисы запущены.

### Шаг 6 — Открыть SignFinder

Откройте браузер и перейдите по адресу: **http://localhost:8501**

Введите ваш **ACCESS_CODE** из файла `.env`.

**Первое что нужно сделать после входа — настроить ИИ-провайдер:**

1. Перейдите в **Настройки → LLM**
2. Выберите провайдера и вставьте API-ключ (инструкция по получению ключа DeepSeek — ниже)

## Часть 2. Настройка DeepSeek (рекомендуется)

DeepSeek — языковая модель, которая анализирует договоры. Рекомендуется как основной провайдер: работает быстро и стоит значительно дешевле альтернатив.

### Получить ключ DeepSeek

1. Перейти на https://platform.deepseek.com
2. Зарегистрироваться (электронная почта + пароль)
3. После входа перейти в раздел **API Keys**
4. Нажать **Create new secret key**
5. Скопировать ключ — он выглядит как `sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`
6. **Сохраните ключ** — после закрытия окна он больше не отобразится

> На новом аккаунте DeepSeek предоставляет бесплатные кредиты для тестирования.

### Подключить DeepSeek в SignFinder

1. Откройте SignFinder → **Настройки → LLM**
2. Выберите провайдера **DeepSeek**
3. Вставьте скопированный ключ в поле API Key
4. Нажмите **Сохранить**
5. Нажмите **Проверить подключение** — должен появиться зелёный статус

## Часть 3. Настройка почтового агента

Почтовый агент автоматически обрабатывает входящие письма с договорами: сам анализирует, подписывает и раскладывает по папкам. Подробнее о том как он работает — в разделе «Как работает SignFinder».

Настройка почты выполняется в интерфейсе: **Настройки → Mail**. Поддерживается два способа подключения.

### Способ 1 — Обычный пароль (Basic Auth)

Подходит для корпоративных почтовых серверов (Exchange on-premise, обычный IMAP).

В разделе **Настройки → Mail** заполните:
- **IMAP-сервер** и порт
- **Логин** (адрес почты)
- **Пароль**
- **SMTP-сервер** (для ответных писем)

> Для Gmail с обычным паролем: сначала включите «Двухфакторную аутентификацию» в аккаунте Google, затем создайте «Пароль приложения» (App Password) в настройках безопасности Google — используйте его вместо обычного пароля.

### Способ 2 — OAuth2 (для Gmail, Microsoft 365, Яндекс и др.)

Более современный и надёжный способ без использования пароля. Подходит если ваш провайдер не принимает пароли напрямую.

В разделе **Настройки → Mail → OAuth2**:
1. Выберите провайдера (Google, Microsoft, Яндекс, Mail.ru, Rambler)
2. Настройки серверов подставятся автоматически
3. Заполните Client ID, Client Secret и Refresh Token

Подробная инструкция по получению этих данных для каждого провайдера — в файле `OAUTH2_SETUP.md` в папке с программой.

## Часть 4. Ежедневная работа

### Запуск и остановка

```
# Запустить SignFinder
docker compose up -d

# Остановить
docker compose down
```

### Обновление программы

```
git pull
docker compose build
docker compose up -d
```

### Резервная копия данных

Все ваши данные (шаблоны, подписи, настройки) хранятся в папке `C:\SignFinder\data\api\`. Для резервной копии — просто скопируйте эту папку.

## Часть 5. Как работает SignFinder

Этот раздел объясняет принципы работы программы простыми словами — без технических деталей.

### Что подаётся на вход и что выходит

**На вход** принимаются договоры в форматах:
- **PDF** — напрямую
- **DOCX** (Word) — автоматически конвертируется в PDF перед обработкой

**На выходе** всегда **PDF** с наложенной подписью в формате изображения (PNG).

Программа не создаёт юридически значимую электронную подпись (ЭЦП) — она размещает изображение вашей рукописной подписи в нужных местах документа.

---

### Как SignFinder находит места подписи

Когда вы загружаете договор, программа идёт по нескольким уровням поиска — от быстрого к медленному:

**Уровень 0 — Шаблон (мгновенно, бесплатно)**
Если этот тип договора уже встречался раньше — SignFinder узнаёт его по «отпечатку» (симхешу) и сразу знает куда ставить подпись. Подробнее о симхеше — ниже.

**Уровень 1 — Поиск по тексту**
Программа ищет в тексте договора характерные маркеры: подчёркивания `____`, точечные линии `......`, слова «Подпись», «Signature», «Podpis», DocuSign-метки и другие признаки места для подписи.

**Уровень 2 — Искусственный интеллект**
Если автоматического поиска недостаточно — документ анализирует языковая модель (DeepSeek или другой провайдер). Она читает шапку договора, определяет кто является сторонами, генерирует точные поисковые паттерны под этот конкретный документ.

**Уровень 3 — Ручная разметка**
Если ИИ не справился — оператор кликает по нужному месту в интерфейсе и сохраняет разметку. В следующий раз такой договор пройдёт по уровню 0.

---

### Привязка к тексту, а не к координатам

Это важное отличие SignFinder от простых PDF-редакторов.

Обычные программы запоминают: «поставь подпись на координатах X=400, Y=700 страницы 5». Если следующий договор немного отличается (добавился абзац, изменился размер шрифта) — подпись окажется не на том месте.

SignFinder запоминает **текст рядом с местом подписи**: «поставь подпись рядом со строкой, содержащей "Генеральный директор" и подчёркивание». Это работает надёжно даже если договор немного изменился — текстовые якоря не «съезжают».

---

### Пайплайн синонимов и сторон договора

Чтобы найти *наше* место подписи (а не место контрагента), программа должна понять кто из сторон — мы.

Для этого настраивается **профиль подписанта** (Настройки → Подписант):
- Название компании на разных языках: «ООО Инлайн технолоджис», «Innowise Sp. z o.o», «Innowise Group»
- Имя подписанта в разных транслитерациях: «Лебедев А.П.», «Vadim Borisov», «Вадим Борисов»
- **Маркеры** — короткие характерные слова по которым программа узнаёт нашу сторону в любом договоре (например «Innowise»)

Когда ИИ анализирует шапку договора, он находит все стороны, сопоставляет их с профилем подписанта и определяет: вот наша сторона — значит нужно искать место подписи рядом с упоминанием именно нашей компании.

Это позволяет правильно работать с договорами где одна компания бывает то Исполнителем, то Заказчиком — роль не важна, важно название.

---

### Двуязычные договоры

Если договор оформлен в двух колонках (левая — один язык, правая — другой), SignFinder автоматически это определяет и ставит подпись в обе языковые колонки.

---

### Обработка изображения подписи

Когда вы загружаете скан своей подписи, программа автоматически:
- Обрезает лишнее белое пространство вокруг подписи
- Удаляет фон (делает его прозрачным)
- Сохраняет только чернила (чёрные и синие оттенки)
- Масштабирует до нужного размера

Это делается один раз при загрузке. Готовая подпись хранится и используется для всех документов.

---

### Шаблоны и симхеш — быстрое повторное подписание

**Шаблон** — это сохранённая информация о том, куда ставить подпись в документе определённого типа. Шаблоны создаются автоматически или вручную оператором.

Но как программа узнаёт что новый документ похож на уже виденный? Для этого используется **симхеш** (simhash) — «отпечаток текста».

Симхеш — это короткое числовое значение, которое вычисляется из содержания документа. Похожие документы дают похожие симхеши. Если новый договор похож на сохранённый шаблон (высокое совпадение симхеша) — программа сразу применяет готовый шаблон и не тратит время на ИИ-анализ.

**Пример:** у вас 50 договоров аренды от одного контрагента. Первый договор ИИ проанализировал за 10 секунд. Остальные 49 обрабатываются по шаблону за 1-2 секунды каждый — без вопросов к ИИ.

Светофор в интерфейсе показывает уверенность программы:
- 🟢 **Зелёный** — шаблон применён, подпись расставлена автоматически
- 🟡 **Жёлтый** — места найдены, но нужна проверка оператором
- 🔴 **Красный** — технический сбой, документ не обработан

---

### Почтовый агент

Агент — фоновый процесс, который сам опрашивает почту и обрабатывает договоры без участия оператора.

**Схема работы:**
1. Контрагент присылает договор на вашу почту в папку `SignfinderIn`
2. Агент проверяет почту каждые 5 минут (настраивается)
3. Файл анализируется и подписывается автоматически
4. Если подпись расставлена уверенно (🟢) — агент сам отправляет ответное письмо с подписанным документом и перекладывает письмо в папку `SignfinderGreen`
5. Если уверенности нет (🟡) — письмо попадает в папку `SignfinderYellow` и очередь разбора в интерфейсе
6. Оператор разбирает жёлтые письма — подтверждает или корректирует

Streamlit (интерфейс) для работы агента **не нужен** — агент работает самостоятельно пока запущен Docker.

## Часть 6. Решение проблем

**SignFinder не открывается в браузере**
→ Убедитесь что Docker Desktop запущен (значок кита в трее зелёный).
→ Подождите 30-60 секунд после `docker compose up -d` и обновите страницу.

**«Неверный код доступа»**
→ Проверьте значение `ACCESS_CODE` в файле `.env`.

**ИИ не работает, ошибки анализа**
→ Проверьте настройки LLM: **Настройки → LLM → Проверить подключение**.
→ Убедитесь что на счету провайдера есть средства.

**Агент почты не подключается**
→ Проверьте настройки: **Настройки → Mail**.
→ Для Gmail убедитесь что используется App Password, а не обычный пароль.

**Остановить и полностью сбросить данные**
```
docker compose down
```
Удалить данные (если нужен полный сброс):
```
rmdir /s /q data\api
```
> Внимание: удалятся все шаблоны, подписи и настройки.

## Часть 7. Структура файлов

```
C:\SignFinder\
├── .env                  ← ваши настройки (не передавайте никому)
├── .env.example          ← шаблон настроек
├── docker-compose.yml    ← конфигурация запуска
├── data\                 ← все ваши данные (создаётся автоматически)
│   └── api\
│       ├── signers\      ← профили подписантов и PNG подписей
│       ├── templates\    ← сохранённые шаблоны договоров
│       └── settings\     ← настройки системы
├── OAUTH2_SETUP.md       ← инструкция по OAuth2 для почты
└── GMAIL_SETUP.md        ← инструкция по настройке Gmail
```
