# SignFinder — Руководство по установке и использованию

SignFinder — система автоматического поиска мест подписи в договорах и проставления факсимиле. Поддерживает PDF и DOCX, работает на вашем компьютере без передачи документов в облако (кроме запросов к языковой модели).

---

## Часть 1. Установка

> Все команды выполняются в **PowerShell от имени администратора**.
> Чтобы открыть: нажмите Win, введите `PowerShell`, правая кнопка мыши → **Запуск от имени администратора**.

### Шаг 1 — Установить Git

Git нужен для скачивания программы с GitHub.

1. Перейти на https://git-scm.com/download/win
2. Скачать установщик (64-bit) и запустить
3. Во всех шагах установщика нажимать **Next** — настройки по умолчанию подходят
4. После установки проверить в PowerShell:
   ```powershell
   git --version
   ```
   Должно появиться: `git version 2.xx.x`

### Шаг 2 — Установить Docker Desktop

Docker — среда, в которой запускается SignFinder. Ничего дополнительно устанавливать не нужно — всё работает внутри Docker.

1. Перейти на https://www.docker.com/products/docker-desktop/
2. Нажать **Download for Windows**, запустить установщик
3. После установки **перезагрузить компьютер**
4. Запустить Docker Desktop — в трее должен появиться значок кита (зелёный = готов)
5. Проверить в PowerShell:
   ```powershell
   docker --version
   ```
   Должно появиться: `Docker version 2x.x.x`

> **Важно:** Docker Desktop должен быть запущен каждый раз, когда вы работаете с SignFinder.

### Шаг 3 — Скачать SignFinder

```powershell
New-Item -ItemType Directory -Path "C:\Program Files\SignFinder" -Force
cd "C:\Program Files\SignFinder"
git clone https://github.com/alexgeorg2507-creator/SignPDFMVPLocal.git .
```

Точка в конце последней команды обязательна — означает «скачать в текущую папку».

### Шаг 4 — Создать файл настроек

```powershell
cd "C:\Program Files\SignFinder"
Copy-Item .env.example .env
```

Откройте файл `.env` в Блокноте:

```powershell
notepad .env
```

Заполните **два обязательных поля**:

```
API_KEY=придумайте_любую_строку_минимум_20_символов
ACCESS_CODE=1234
```

**API_KEY** — внутренний пароль системы. Придумайте любую строку из букв и цифр длиной не менее 20 символов. Пример: `mySuperLocalKey_20260601_abc`. Вы его вводить не будете — он защищает внутренние запросы.

**ACCESS_CODE** — код для входа в интерфейс. Оставьте `1234` или замените на свой.

Сохраните файл и закройте.

> Ключ языковой модели (ИИ) вводится через интерфейс программы после запуска — см. Шаг 6.

### Шаг 5 — Собрать и запустить

```powershell
cd "C:\Program Files\SignFinder"
docker compose build
```

Первый запуск занимает **15–25 минут** — Docker скачивает все компоненты. Последующие запуски занимают секунды.

После завершения сборки:

```powershell
docker compose up -d
```

Флаг `-d` означает «запустить в фоне». Дождитесь сообщения что все сервисы запущены.

### Шаг 6 — Открыть SignFinder

Откройте браузер: **http://localhost:8501**

Введите ваш **ACCESS_CODE** из файла `.env`.

**Первое что нужно сделать — настроить ИИ-провайдер:**

Перейдите в **Настройки → LLM** → выберите провайдера → вставьте API-ключ (инструкция по DeepSeek — ниже).

### Ежедневная работа

```powershell
# Запустить SignFinder
cd "C:\Program Files\SignFinder"
docker compose up -d

# Остановить
docker compose down

# Обновить программу
git pull
docker compose build
docker compose up -d
```

## Часть 2. Настройка DeepSeek (рекомендуется)

DeepSeek — языковая модель, которая анализирует договоры. Рекомендуется как основной провайдер: работает быстро и значительно дешевле альтернатив.

**Ориентировочная стоимость:** около $5 на 1000 договоров при полном анализе.

### Получить ключ DeepSeek

1. Перейти на https://platform.deepseek.com
2. Зарегистрироваться (email + пароль)
3. После входа — раздел **API Keys → Create new secret key**
4. Скопировать ключ вида `sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`
5. **Сохраните ключ** — после закрытия окна он больше не отобразится
6. **Пополните счёт** — без средств на балансе запросы не будут выполняться. Рекомендуем положить $5–10 для начала (Billing → Top up)

### Подключить DeepSeek в SignFinder

Откройте SignFinder → **Настройки → LLM**:
1. Выберите провайдера **DeepSeek**
2. Вставьте скопированный ключ
3. Нажмите **Сохранить** → **Проверить подключение** (должен появиться зелёный статус)

## Часть 3. Настройка почтового агента

Почтовый агент автоматически обрабатывает входящие письма с договорами. Настройка выполняется в интерфейсе: **Настройки → Mail**.

### Папки IMAP

Перед настройкой создайте в вашем почтовом ящике следующие папки (можно через веб-интерфейс почты, в GMAIL папки называются ярлыками. не располагайте папки друг под другом, просто переченть в INBOx):

| Папка | Назначение |
|-------|-----------|
| `SignfinderIn` | Сюда кладёте письма с договорами для подписания |
| `SignfinderGreen` | Договор подписан автоматически, ответ отправлен |
| `SignfinderYellow` | Требуется проверка оператором |
| `SignfinderRed` | Письмо отклонено оператором |
| `SignfinderArchive` | Обработанные письма |

**Что происходит с письмом:**
- Агент нашёл место подписи уверенно 🟢 → подписывает → отправляет ответ контрагенту(опция) → перекладывает в `SignfinderGreen`
- Нашёл место, но не уверен 🟡 → подпись проставлена предварительно → письмо в `SignfinderYellow` → оператор проверяет в разделе **Агент Mail**
- Оператор нажал **Подтвердить** → письмо уходит в `SignfinderGreen`, ответ отправляется
- Оператор нажал **Отклонить** → письмо перекладывается в `SignfinderRed`, ответ не отправляется

### Способ 1 — Обычный пароль (Basic Auth)

Подходит для корпоративных серверов (Exchange on-premise, обычный IMAP).

В **Настройки → Mail** заполните:
- IMAP-сервер и порт (обычно 993)
- Логин (адрес почты)
- Пароль
- SMTP-сервер для ответных писем

> **Для Gmail:** обычный пароль не работает. Используйте App Password (пароль приложения) или OAuth2. Для App Password: включите двухфакторную аутентификацию → Google Account → Безопасность → Пароли приложений → создайте пароль для «Другое» → используйте вместо обычного пароля.

### Способ 2 — OAuth2 (для Gmail, Microsoft 365, Яндекс и др.)

Современный способ без пароля. Используйте если ваш провайдер не принимает пароли напрямую, или если хотите более надёжную защиту.

В **Настройки → Mail → OAuth2** выберите провайдера — настройки серверов подставятся автоматически. Нужны три значения: Client ID, Client Secret, Refresh Token.

**Подробная инструкция для Gmail — ниже.**

## Часть 4. Настройка Gmail через OAuth2 — пошаговая инструкция

> Эта инструкция основана на реальном опыте настройки. Следуйте шагам точно в указанном порядке.

### Шаг 1 — Создать проект в Google Cloud

1. Откройте https://console.cloud.google.com и войдите под своим аккаунтом Gmail
2. Вверху нажмите дропдаун проектов → **New Project**
3. Назовите проект (например `signfinder`) → **Create**
4. После создания: **APIs & Services → Library**
5. Найдите **Gmail API** → нажмите **Enable**

### Шаг 2 — Настроить OAuth consent screen

1. **APIs & Services → OAuth consent screen**
2. User Type: **External** → **Create**
3. Заполните:
   - App name: `SignFinder`
   - User support email: ваш Gmail
   - Developer contact email: ваш Gmail
4. Нажимайте **Save and Continue** на всех шагах до конца
5. На шаге **Test users** → нажмите **+ Add users** → добавьте ваш Gmail-адрес → **Save**

> **Важно:** если не добавить себя в тестовые пользователи — при запуске скрипта получите ошибку `Error 403: access_denied`. Этот шаг обязателен.

### Шаг 3 — Создать OAuth Client ID

1. **APIs & Services → Credentials → + Create Credentials → OAuth client ID**
2. Application type: **Desktop app**
3. Name: `signfinder-desktop` (любое)
4. **Create**
5. Появится окно с двумя значениями — скопируйте оба:
   ```
   Client ID:     xxxxxxxxxx-xxxxxxxx.apps.googleusercontent.com
   Client secret: GOCSPX-xxxxxxxxxxxxxxxxxxxxxxxxxx
   ```

### Шаг 4 — Получить Refresh Token

Откройте PowerShell **на вашем компьютере** (не внутри Docker) и перейдите в папку SignFinder:

```powershell
cd "C:\Program Files\SignFinder"
```

Запустите скрипт **одной строкой** (замените значения на свои):

```powershell
python get_refresh_token.py --provider google --client-id "ВАШ_CLIENT_ID.apps.googleusercontent.com" --client-secret "ВАШ_GOCSPX_СЕКРЕТ"
```

> **Важно:** команда должна быть в одну строку. Многострочный ввод в PowerShell через перенос строки не работает.

После запуска:
1. Автоматически откроется браузер с экраном разрешений Google
2. Нажмите **Продолжить** → разрешите доступ
3. В терминале PowerShell появится **Refresh Token** — скопируйте его

### Шаг 5 — Ввести данные в SignFinder

Откройте SignFinder → **Настройки → Mail → OAuth2 (XOAUTH2) → Google**:

| Поле | Значение |
|------|---------|
| User | ваш Gmail-адрес |
| Client ID | из Шага 3 |
| Client Secret | из Шага 3 |
| Refresh Token | из Шага 4 |

Нажмите **Сохранить**. В логах агента (Агент Mail → журнал) появится:
```
IMAP XOAUTH2 logged in as ваш@gmail.com (google)
```

## Часть 5. Настройка профиля подписанта

После первого запуска нужно настроить кто именно ставит подпись.

Перейдите в **Настройки → Подписант**:
1. Выберите профиль из списка или создайте новый
2. Укажите отображаемое имя
3. Добавьте маркеры — короткие характерные слова по которым программа узнаёт вашу компанию в договоре (например «Innowise» или «Инлайн технолоджис»)
4. Добавьте алиасы компании и подписанта на всех нужных языках
5. Загрузите изображение подписи (PNG, JPG или GIF)

Программа автоматически обработает изображение: уберёт фон, обрежет лишнее белое пространство, оптимизирует размер.

## Часть 6. Как работает SignFinder

### Что подаётся на вход и что выходит

**На вход** принимаются договоры в форматах:
- **PDF** — напрямую
- **DOCX** (Word) — автоматически конвертируется в PDF перед обработкой

**На выходе** всегда **PDF** с наложенным факсимиле в виде изображения.

> SignFinder не создаёт юридически значимую электронную подпись (ЭЦП) — он размещает изображение вашего факсимиле в нужных местах документа.

---

### Как SignFinder находит места подписи

Программа идёт по нескольким уровням поиска — от быстрого к медленному:

**Уровень 0 — Шаблон (мгновенно, бесплатно)**
Если этот тип договора уже встречался — SignFinder узнаёт его по «отпечатку» и сразу знает куда ставить факсимиле. Подробнее — в разделе про симхеш.

**Уровень 1 — Поиск по тексту**
Программа ищет характерные маркеры мест подписи:
- Подчёркивания `____`
- Точечные линии `......`
- Слова «Подпись», «Signature», «Podpis», «Потпис»
- DocuSign-метки (`\s1\`, `\e1\`, `\t1\` — специальные теги системы DocuSign, вставляемые в документ)

**Уровень 2 — Искусственный интеллект**
Если автоматического поиска недостаточно — договор анализирует языковая модель. Она читает шапку договора, определяет стороны, генерирует точные поисковые паттерны под этот конкретный документ.

**Уровень 3 — Ручная разметка**
Оператор кликает по нужному месту в интерфейсе и сохраняет шаблон. Следующий раз такой договор пройдёт по Уровню 0, а подпись всегда будет стоять там же (даже если в договоре изменится текст)

---

### Привязка к тексту, а не к координатам

Обычные программы запоминают: «поставь факсимиле на координатах X=400, Y=700 страницы 5». Если следующий договор немного отличается (добавился абзац, изменился шрифт) — факсимиле окажется не в том месте.

SignFinder запоминает **текст рядом с местом подписи**: «поставь факсимиле рядом со строкой "Генеральный директор" и подчёркиванием». Это работает надёжно даже если договор изменился — текстовые якоря не сдвигаются.

---

### Пайплайн синонимов и сторон договора

Чтобы найти *наше* место подписи (а не место контрагента), программа должна понять кто из сторон — мы.

В **профиле подписанта** настраиваются:
- Название компании на разных языках: «ООО Инлайн технолоджис», «Innowise Sp. z o.o», «Innowise Group»
- Имя подписанта в разных транслитерациях: «Лебедев А.П.», «Vadim Borisov», «Вадим Борисов»
- **Маркеры** — короткие слова по которым программа однозначно узнаёт нашу компанию (например «Innowise»)

Когда ИИ анализирует шапку договора — он находит все стороны, сопоставляет с профилем и определяет нашу сторону. Это работает независимо от роли в договоре: ваша компания может быть то Исполнителем, то Заказчиком — программа ориентируется на название, не на роль.

---

### Двуязычные договоры

Если договор оформлен в двух колонках (левая — один язык, правая — другой), SignFinder автоматически определяет это и ставит факсимиле в обе языковые колонки.

---

### Обработка факсимиле

При загрузке изображения подписи программа автоматически:
- Убирает фон (делает его прозрачным)
- Обрезает лишнее белое пространство
- Сохраняет только чернила (чёрные и синие оттенки)
- Масштабирует до нужного размера (~15мм высота)

Это делается один раз при загрузке. Готовое факсимиле хранится и используется для всех документов.

---

### Шаблоны и симхеш — быстрое повторное подписание

**Шаблон** — сохранённая информация о том, куда ставить факсимиле в документе определённого типа.

**Симхеш** (simhash) — «текстовый отпечаток» документа. Короткое числовое значение, вычисленное из содержания договора. Похожие документы дают похожие симхеши.

Если новый договор похож на сохранённый шаблон — программа сразу применяет готовый шаблон, не тратя время на ИИ.

**Пример:** у вас 50 договоров аренды от одного контрагента. Первый договор ИИ проанализировал за 10 секунд. Остальные 49 обрабатываются по шаблону за 1–2 секунды каждый.

Светофор показывает уверенность программы:
- 🟢 **Зелёный** — шаблон применён, факсимиле расставлено автоматически
- 🟡 **Жёлтый** — места найдены, рекомендуется проверка оператором
- 🔴 **Красный** — технический сбой, документ не обработан

---

### Почтовый агент

Агент — фоновый процесс, который сам опрашивает почту и обрабатывает договоры без участия оператора.

**Схема работы:**
1. Контрагент (или вы сами с помощью почтовых правил) присылает договор в папку `SignfinderIn` вашего ящика
2. Агент проверяет почту каждые 5 минут (настраивается)
3. Если факсимиле расставлено уверенно 🟢 — агент сам отправляет ответ и перекладывает письмо в `SignfinderGreen`
4. Если нет уверенности 🟡 — письмо попадает в `SignfinderYellow` и в очередь разбора. В случае, если система видит договор первый раз, шаблон на нее не создан - уверенности не будет
5. Оператор разбирает жёлтые письма в разделе **Агент Mail**:
   - **Подтвердить** → ответ отправляется, письмо в `SignfinderGreen`
   - **Отклонить** → письмо в `SignfinderRed`, ответ не отправляется
   - **В разбор** → открывается редактор разметки

Интерфейс Streamlit для работы агента **не нужен** — агент работает самостоятельно пока запущен Docker.

## Часть 7. Решение проблем

**SignFinder не открывается в браузере**
→ Убедитесь что Docker Desktop запущен (значок кита в трее зелёный).
→ Подождите 30–60 секунд после запуска и обновите страницу.

**«Неверный код доступа»**
→ Проверьте `ACCESS_CODE` в файле `.env`.

**ИИ не работает**
→ Настройки → LLM → Проверить подключение.
→ Проверьте баланс на счету провайдера.

**Агент почты не подключается**
→ Настройки → Mail → проверьте настройки.
→ Для Gmail: убедитесь что используется App Password или настроен OAuth2.

**Gmail OAuth2: ошибка `access_denied`**
→ Вы не добавили себя в тестовые пользователи. Откройте Google Cloud Console → APIs & Services → OAuth consent screen → Test users → добавьте ваш Gmail.

**Остановить SignFinder**
```powershell
cd "C:\Program Files\SignFinder"
docker compose down
```

**Полный сброс данных (осторожно — удалятся все шаблоны и настройки)**
```powershell
cd "C:\Program Files\SignFinder"
Remove-Item -Recurse -Force "data\api"
```

## Часть 8. Резервная копия и перенос

Все данные (шаблоны, факсимиле, настройки) хранятся в папке `C:\Program Files\SignFinder\data\api\`.

**Создать резервную копию:**
```powershell
Copy-Item -Recurse "C:\Program Files\SignFinder\data\api" "C:\Backup\signfinder-data-backup"
```

**Перенести на другой компьютер:**
1. Установить SignFinder на новом компьютере (Шаги 1–5)
2. Скопировать папку `data\api\` с прежнего компьютера в `C:\Program Files\SignFinder\data\api\`
3. Запустить `docker compose up -d`

Все шаблоны, факсимиле и настройки будут восстановлены.
