# SignFinder — Настройка почтовых ящиков для агента

Агент опрашивает папку `SignfinderIn`, обрабатывает договора из вложений,
раскладывает по светофору. **`SignfinderIn` — не инбокс**, а отфильтрованные
договорные письма. Фильтрация настраивается на стороне почты (эта инструкция).

Папки агента (5 штук):
```
SignfinderIn       ← входящие договора (агент опрашивает)
SignfinderGreen    ← подписано уверенно
SignfinderYellow   ← подписано, на проверку
SignfinderRed      ← сбой / мусор / отклонено
SignfinderArchive  ← оригиналы после обработки
```

Настройки подключения вводятся в UI: **Настройки → Mail**.

---

## 1. Gmail / Google Workspace

В Gmail вместо папок — **ярлыки (labels)**, по IMAP видны как папки.

### 1.1 Создать ярлыки

Gmail → Настройки (⚙️) → «Все настройки» → вкладка «Ярлыки» → «Создать ярлык».
Создать 5 плоских ярлыков (БЕЗ вложенности):
`SignfinderIn`, `SignfinderGreen`, `SignfinderYellow`, `SignfinderRed`, `SignfinderArchive`.

> Плоские ярлыки (не `SignFinder/In`) — чтобы избежать путаницы с IMAP-разделителями.

### 1.2 Фильтр для договоров → SignfinderIn

Gmail → Настройки → «Фильтры и заблокированные адреса» → «Создать фильтр».

Критерий (выбрать под свой случай):
- `has:attachment filename:pdf` — письма с PDF
- `from:(контрагент@домен.ru)` — от конкретных отправителей
- `subject:(договор OR contract OR umowa)` — по теме

Действие фильтра:
- ☑ Применить ярлык: **SignfinderIn**
- ☑ Пропустить «Входящие» (Архивировать) — чтобы не дублировалось в инбоксе

### 1.3 Включить IMAP

Gmail → Настройки → «Пересылка и POP/IMAP» → «Доступ по IMAP: Включить».

### 1.4 Аутентификация — App Password

Gmail не принимает обычный пароль по IMAP. Нужен пароль приложения:

1. Включить 2FA: Аккаунт Google → Безопасность → Двухэтапная аутентификация
2. Аккаунт Google → Безопасность → «Пароли приложений»
3. Создать пароль для «Почта» → скопировать 16-значный код
4. Этот код вводится в Настройки → Mail как **IMAP password** (не основной пароль аккаунта)

### 1.5 Параметры в UI (Настройки → Mail)

```
IMAP Host: imap.gmail.com
IMAP Port: 993
IMAP SSL:  ✓
IMAP User: you@gmail.com
IMAP Pass: <App Password 16 знаков>

SMTP Host: smtp.gmail.com
SMTP Port: 587
SMTP User: you@gmail.com
SMTP Pass: <тот же App Password>

Папки: SignfinderIn / Green / Yellow / Red / Archive
```

## 2. Microsoft Exchange / Outlook / Office 365

Два варианта в зависимости от типа аккаунта.

### 2.1 Outlook.com (личный) — папки

В Outlook папки настоящие (не ярлыки).

1. Outlook.com → правый клик на «Папки» → «Создать папку»
2. Создать 5 папок: `SignfinderIn`, `SignfinderGreen`, `SignfinderYellow`, `SignfinderRed`, `SignfinderArchive`
3. Правило: Настройки → «Почта» → «Правила» → «Добавить правило»
   - Условие: есть вложение / от отправителя / тема содержит «договор»
   - Действие: «Переместить в» → SignfinderIn

### 2.2 IMAP-доступ для Outlook.com

Настройки → «Почта» → «Синхронизация электронной почты» → включить IMAP (POP и IMAP).

Параметры:
```
IMAP Host: outlook.office365.com
IMAP Port: 993
IMAP SSL:  ✓
SMTP Host: smtp.office365.com
SMTP Port: 587
```

### 2.3 Аутентификация — ЗАСАДА с Office 365

Microsoft отключил basic auth (логин+пароль) для Exchange Online с 2022.
Варианты:

**A. App Password (если включена MFA и разрешено админом)**
- account.microsoft.com → Безопасность → «Дополнительные параметры безопасности»
- «Пароли приложений» → создать → использовать как IMAP password
- Работает НЕ во всех тенантах — админ может запретить

**B. Corporate Exchange (on-prem или гибрид)**
- Если у клиента свой Exchange-сервер (не облако) — basic auth обычно ещё работает
- IMAP host = адрес их Exchange (узнать у админа)
- Логин = обычный, или DOMAIN\username

**C. OAuth2 (правильный путь для O365)**
- В v1.16 НЕ реализован — только basic auth / app password
- Планируется позже как отдельный IntakeSource-адаптер (Microsoft Graph API)

### 2.4 Если basic auth заблокирован тенантом

Для корпоративного O365 где админ запретил app passwords и basic auth —
агент в текущей версии (v1.16, basic auth) НЕ подключится. Нужен один из:
- Попросить админа разрешить app password для сервисного ящика
- Использовать выделенный почтовый ящик на сервере с basic auth
- Дождаться OAuth2 / Graph API адаптера (roadmap)

## 3. Универсальный IMAP (любой другой сервер)

Postfix, Dovecot, Zoho, Yandex, Mail.ru и др. — стандартный IMAP:

1. Создать 5 папок через любой почтовый клиент
2. Настроить серверное правило: договорные письма → SignfinderIn
3. Узнать IMAP/SMTP host и port у провайдера
4. Ввести креды в Настройки → Mail

| Провайдер | IMAP host | Port |
|-----------|-----------|------|
| Yandex | imap.yandex.ru | 993 |
| Mail.ru | imap.mail.ru | 993 |
| Zoho | imap.zoho.com | 993 |

Яндекс/Mail.ru тоже требуют пароль приложения при включённой 2FA.

---

## 4. Проверка после настройки

1. Настройки → Mail → ввести креды → «🔍 Тест IMAP» → должно быть OK
2. Положить тестовый договор в SignfinderIn (письмо с PDF)
3. Агент Mail → «🔄 Опросить почту сейчас»
4. Письмо должно уйти в Green/Yellow/Red, оригинал — в Archive
5. Журнал агента — запись о обработке

## 5. Резюме по аутентификации (главные грабли)

| Провайдер | Basic auth | App Password | OAuth2 (v1.16) |
|-----------|-----------|--------------|----------------|
| Gmail | ❌ убит | ✅ нужен 2FA | ⏳ позже |
| Outlook.com | ⚠️ частично | ✅ обычно | ⏳ позже |
| O365 корпоративный | ❌ убит 2022 | ⚠️ если админ разрешил | ⏳ позже |
| Exchange on-prem | ✅ работает | — | — |
| Yandex/Mail.ru | ⚠️ при 2FA нет | ✅ | — |

**Вывод:** для облачных Gmail/O365 — App Password. Для on-prem Exchange —
обычный basic auth. OAuth2 для самых строгих тенантов — в планах.
